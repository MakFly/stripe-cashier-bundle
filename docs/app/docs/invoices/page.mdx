# Factures

Le bundle couvre deux aspects différents:

1. la lecture des invoices Stripe
2. l'archivage local d'un PDF généré dans `var/data/invoices`

## Lire les invoices Stripe

```php
use CashierBundle\Service\InvoiceService;

$invoices = $invoiceService->list($user);
$invoice = $invoiceService->find('in_xxx');
$upcoming = $invoiceService->upcoming($user);
```

## Télécharger un PDF à la demande

```php
$response = $invoice->download([
    'company_name' => 'SF Cashier',
    'company_email' => 'billing@example.test',
    'locale' => 'fr',
]);
```

Le renderer par défaut sait:

- générer le PDF binaire
- retourner une `Response` HTTP
- streamer le PDF inline

## Archivage automatique

Lorsqu'un paiement Stripe déclenche `invoice.paid`, le bundle:

1. récupère l'invoice Stripe
2. construit la vue de facture
3. rend le PDF
4. stocke le fichier dans `var/data/invoices`
5. persiste la trace dans `cashier_generated_invoices`

## `GeneratedInvoice`

La table d'archivage stocke notamment:

- `stripeInvoiceId`
- `stripePaymentIntentId`
- `stripeCheckoutSessionId`
- `filename`
- `relativePath`
- `currency`
- `total`
- `payload`

Le `payload` archive aussi les `metadata` Stripe utiles pour rapprocher la facture d'une commande métier.

## Internationalisation

Le provider par défaut supporte `en` et `fr`.

Priorité de langue:

1. `locale` ou `invoice_locale` passé au rendu
2. `preferred_locales` du customer Stripe
3. `cashier.invoices.default_locale`

## Sans `ext-intl`

Le bundle reste utilisable.

Fallbacks:

- dates en `YYYY-MM-DD`
- format monétaire simplifié
- labels traduits conservés
