# Exceptions

Le bundle définit plusieurs exceptions spécifiques pour gérer les erreurs liées à Stripe et aux opérations de facturation.

## Liste des exceptions

### CustomerAlreadyCreatedException

Exception levée lorsqu'on tente de créer un client Stripe qui existe déjà.

#### Emplacement

```php
namespace CashierBundle\Exception;

class CustomerAlreadyCreatedException extends Exception
```

#### Méthodes statiques

```php
// Crée une exception avec l'ID Stripe du client existant
CustomerAlreadyCreatedException::create(string $stripeId): self
```

#### Exemple d'utilisation

```php
try {
    $user->createAsStripeCustomer();
} catch (CustomerAlreadyCreatedException $e) {
    // Le client existe déjà
    $stripeId = $user->stripeId();
}
```

---

### IncompletePaymentException

Exception levée lorsqu'un paiement nécessite une action supplémentaire (authentification 3D Secure, etc.).

#### Emplacement

```php
namespace CashierBundle\Exception;

class IncompletePaymentException extends Exception
```

#### Méthodes

```php
// Récupère l'objet Payment associé
$exception->payment(): Payment
```

#### Exemple d'utilisation

```php
use CashierBundle\Exception\IncompletePaymentException;

try {
    $payment = $user->charge(1000, 'pm_abc123');
} catch (IncompletePaymentException $e) {
    $payment = $e->payment();

    if ($payment->requiresAction()) {
        // Rediriger l'utilisateur vers la page d'authentification
        return new JsonResponse([
            'requires_action' => true,
            'payment_intent_client_secret' => $payment->clientSecret(),
        ]);
    }
}
```

#### Scénarios courants

- Authentification 3D Secure requise
- Moyen de paiement invalide
- Confirmation supplémentaire nécessaire

### InvalidCouponException

Exception levée lorsqu'un coupon ou code promotionnel est invalide.

#### Emplacement

```php
namespace CashierBundle\Exception;

class InvalidCouponException extends Exception
```

#### Méthodes statiques

```php
// Crée une exception avec l'ID du coupon invalide
InvalidCouponException::invalid(string $couponId): self
```

#### Exemple d'utilisation

```php
use CashierBundle\Exception\InvalidCouponException;

try {
    $subscription = $user->newSubscription('default', 'price_monthly')
        ->withCoupon('WRONG_CODE')
        ->create();
} catch (InvalidCouponException $e) {
    // Afficher un message d'erreur à l'utilisateur
    $this->addFlash('error', 'Le code promo est invalide ou a expiré.');
}
```

### InvalidCustomerException

Exception levée lorsque les informations du client sont invalides.

#### Emplacement

```php
namespace CashierBundle\Exception;

class InvalidCustomerException extends Exception
```

#### Méthodes statiques

```php
// Client pas encore créé dans Stripe
InvalidCustomerException::notYetCreated(): self

// ID Stripe invalide
InvalidCustomerException::invalidId(string $stripeId): self
```

#### Exemple d'utilisation

```php
use CashierBundle\Exception\InvalidCustomerException;

try {
    $user->updateStripeCustomer(['email' => 'new@email.com']);
} catch (InvalidCustomerException $e) {
    // Le client n'existe pas dans Stripe
    $user->createAsStripeCustomer();
}
```

### InvalidInvoiceException

Exception levée lors d'une erreur de facturation.

#### Emplacement

```php
namespace CashierBundle\Exception;

class InvalidInvoiceException extends Exception
```

#### Méthodes statiques

```php
// Facture invalide
InvalidInvoiceException::invalid(string $invoiceId): self

// La facture n'appartient pas au client
InvalidInvoiceException::notBelongToCustomer(
    string $invoiceId,
    string $customerId
): self
```

#### Exemple d'utilisation

```php
use CashierBundle\Exception\InvalidInvoiceException;

try {
    $invoice = $user->invoice();
} catch (InvalidInvoiceException $e) {
    $this->logger->error('Erreur de facturation', [
        'error' => $e->getMessage(),
    ]);
}
```

### SubscriptionUpdateFailureException

Exception levée lors de l'échec de la mise à jour d'un abonnement.

#### Emplacement

```php
namespace CashierBundle\Exception;

class SubscriptionUpdateFailureException extends Exception
```

#### Méthodes statiques

```php
// Crée une exception avec une raison personnalisée
SubscriptionUpdateFailureException::create(string $reason): self
```

#### Exemple d'utilisation

```php
use CashierBundle\Exception\SubscriptionUpdateFailureException;

try {
    $subscription = $user->subscription('default');
    $subscriptionService->swap($subscription, 'price_premium');
} catch (SubscriptionUpdateFailureException $e) {
    $this->logger->error('Échec de la mise à jour de l\'abonnement', [
        'error' => $e->getMessage(),
    ]);

    $this->addFlash('error', 'Impossible de modifier l\'abonnement.');
}
```

#### Scénarios courants

- Erreur de communication avec l'API Stripe
- Prix invalide
- Abonnement dans un état qui ne permet pas la modification
- Moyen de paiement défaillant

## Gestion globale des exceptions

### Exemple de gestion dans un contrôleur

```php
use CashierBundle\Exception\{
    CustomerAlreadyCreatedException,
    IncompletePaymentException,
    InvalidCouponException,
    InvalidPaymentMethodException,
    SubscriptionUpdateFailureException
};

class SubscriptionController extends AbstractController
{
    public function create(Request $request): Response
    {
        try {
            $subscription = $this->getUser()->newSubscription('default', 'price_monthly')
                ->trialDays(14)
                ->create($request->request->get('payment_method'));

            return $this->redirectToRoute('subscription_success');

        } catch (CustomerAlreadyCreatedException $e) {
            // Ne devrait pas arriver avec createOrGetStripeCustomer
            return $this->redirectToRoute('subscription_dashboard');

        } catch (InvalidCouponException $e) {
            $this->addFlash('error', 'Code promo invalide.');
            return $this->redirectToRoute('subscription_checkout');

        } catch (InvalidPaymentMethodException $e) {
            $this->addFlash('error', 'Moyen de paiement refusé.');
            return $this->redirectToRoute('payment_setup');

        } catch (IncompletePaymentException $e) {
            return $this->handleIncompletePayment($e->payment());
        }
    }

    private function handleIncompletePayment(Payment $payment): Response
    {
        if ($payment->requiresAction()) {
            return new JsonResponse([
                'requires_action' => true,
                'payment_intent_client_secret' => $payment->clientSecret(),
            ]);
        }

        $this->addFlash('error', 'Paiement incomplet.');
        return $this->redirectToRoute('subscription_checkout');
    }
}
```

### Exemple de gestion via un écouteur d'exceptions

```php
use Symfony\Component\HttpKernel\Event\ExceptionEvent;
use CashierBundle\Exception\InvalidCouponException;
use CashierBundle\Exception\InvalidPaymentMethodException;

class CashierExceptionListener
{
    public function __invoke(ExceptionEvent $event): void
    {
        $exception = $event->getThrowable();

        if ($exception instanceof InvalidCouponException) {
            // Convertir en une réponse JSON pour les requêtes AJAX
            if ($event->getRequest()->isXmlHttpRequest()) {
                $event->setResponse(new JsonResponse([
                    'error' => 'invalid_coupon',
                    'message' => 'Le code promo est invalide.',
                ], 400));
            }
        }

        if ($exception instanceof InvalidPaymentMethodException) {
            if ($event->getRequest()->isXmlHttpRequest()) {
                $event->setResponse(new JsonResponse([
                    'error' => 'invalid_payment_method',
                    'message' => 'Moyen de paiement invalide.',
                ], 400));
            }
        }
    }
}
```

### Configuration de l'écouteur

```yaml
# config/services.yaml
services:
    App\Listener\CashierExceptionListener:
        tags:
            - { name: kernel.event_listener, event: kernel.exception }
```
